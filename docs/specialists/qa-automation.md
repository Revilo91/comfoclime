# Automatisierter Tester (QA)

## Ziel
Sorgt fuer solide Testabdeckung und fruehe Fehlererkennung.

## Verantwortlichkeiten
- Tests aktualisieren bei jeder Codeaenderung
- Mocking-Strategien fuer lokale API-Endpunkte
- Edge-Case-Tests fuer Parser, Cache, Rate-Limits

## Fokusbereiche
- `tests/` (API, Coordinator, Entities)
- Sensor/Property Definition Tests
- Timeout/Retry/Cache Tests

## Deliverables
- Neue Tests fuer neue Funktionen
- Erweiterte Edge-Case-Abdeckung
- Test-Doku oder Hinweise zu Mocks

## Checkliste
- [ ] Tests fuer geaenderte Codepfade
- [ ] Edge-Case-Tests vorhanden
- [ ] Mocking konsistent
- [ ] Testlauf dokumentiert
